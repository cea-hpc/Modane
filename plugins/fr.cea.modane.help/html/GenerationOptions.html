Options de génération

Il existe des options liées à la génération de code. Ces options sont positionnées dans un fichier nommé 'modane.properties'. Elles sont :

  • parallelLoops = true/false : génère des boucles multi-thread pour toutes les fonctions du modèle qui ont un support et dont l'option 'parallel' vaut 'true'. Par défaut, cette option est à vrai et elle peut-être désactivée, par exemple, pour des besoins de débogage.

  • alwaysMaterialMng = true/false : génère des classes avec un accès au gestionnaire de matériaux de Arcane, même si l'entité correspondante dans le modèle (module, service...) ne comporte ni variable, ni support de type matériau. Par défaut cette option est activée.

  • variableAsArgs = true/false : pour chaque fonction du modèle, génère une classe contenant toutes les variables de cette fonction et passe un argument du type de cette classe à la fonction. Par défaut cette option est activée.

  • defaultService = true/false : génère un service/une classe par défaut pour une interface n'ayant aucune implémentation. Par défaut cette option est activée.

  • traceMsg = true/false : génère des messages de trace dans les méthodes des classes de base. Par défaut cette option n'est pas active et elle peut-être activée, par exemple, pour des besoins de débogage.

  • generateXsd = true/false : génère un XML schéma pour chaque élément Modane (Module, Service, Struct...) et un schéma global.xsd représentant l'ensemble du modèle. Par défaut cette option n'est pas active.

  • langFr = true/false : cette option n'est utile que pour la génération des schémas. Si elle vaut vrai, les schémas seront créés avec les noms français des propriétés sinon avec les noms anglais. La valeur par défaut est 'false'.

Les fichiers 'modane.properties' doivent se trouver sur l'arborescence de génération. Quand un package est généré, les fichiers de configuration se trouvant dans son arborescence sont lus dans leur ordre d'apparition depuis la racine.

Par exemple, supposons qu'un package a.b.c soit généré dans src, et que les répertoires 'src' et 'src/a/b' contiennent un fichier 'modane.properties'. Dans ce cas, les options sont positionnées avec le fichier se trouvant dans 'src' et sont ensuite écrasées/complétées avec le fichier se trouvant dans 'src/a/b'.

Ainsi, il est possible de fournir des options pour le projet et de surcharger certaines d'entre elles pour certains packages du modèle.
